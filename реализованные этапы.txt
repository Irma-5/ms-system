Сценарии использования:


Работа системы:

1. ПОДГОТОВКА ДАННЫХ:
   User → WebMaster → Collector → Storage
   - Выбор размера и стратегии батча
   - Collector загружает batch_0, создаёт выборку
   - Сохраняет в Storage как batch__id=N__...pkl

2. ОБУЧЕНИЕ:
   - Модели обучены заранее и загружены в storage/models/ как .pkl.gz

3. ПРИМЕНЕНИЕ:
   User → WebMaster → MLService → Storage
   - Выбор модели и батча
   - MLService загружает модель и данные (с storage)
   - model.predict(X) → survival curves
   - Сохраняет результаты в Storage

4. ОЦЕНКА:
   User → WebMaster → MLService → Storage (→ MLService → Storage)
   - Вычисление IBS, AUPRC
   - Сохранение метрик

5. ВИЗУАЛИЗАЦИЯ:
   User → WebMaster → Visualization → Storage
   - Построение графиков

Реализованные этапы:

1. Архитектура и модульность (до 20 баллов):
	(a) Обязательная часть (10 баллов): наличие независимых сервисов, Dockerfile,
	    docker-compose, четкая структура;

2. Collector (до 10 баллов):
	(a) Обязательная часть (5 баллов): API получения батча данных 
	    (для эмуляции достаточно подавать последовательные части заранее заданного набора), обработка ошибок;
	(b) ii. Расширенный лог действий и запросов (запись в файл);
3. Storage (до 10 баллов):
	(a) Обязательная часть (5 баллов): API CRUD операций базы данных;
	(b) ii. Расширенный лог действий и запросов (запись в файл);
4. ML Service (до 20 баллов):
	(a) Обязательная часть (10 баллов): API запросов создания, обучения,
	    применения;
	(b) Дополнительная часть:
	    i. Сложность модели;
5. Web Master (до 10 баллов):
	(a) Обязательная часть (5 баллов): API сценариев использования;

6. Visualization (до 20 баллов):
	(a) Обязательная часть (10 баллов): интерактивный GUI (прослойка между
	    пользователем и Web Master);
	(b) Дополнительная часть:
	    i. Сложность GUI: веб-сайт, графики;

7. Документирование проекта (до 10 баллов):
	(a) Добавление раздела в README с инструкциями по развертыванию;
	(b) Текстовое описание поставленной задачи, сценариев использования 
	    и аргументация выбранного подхода;
	(c) Разработка средств запуска и управления системой.
